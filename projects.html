  • Project 2

    Video Forgery Detection using CNN & RNN

    1. Hybrid CNN-RNN Architecture: Combines CNNs and RNNs to capture both spatial and temporal features.

    2. Feature Extraction:

    # Xception: 71-layer CNN for spatial features, using 299 x 299 pixel inputs.

    # Inception-v3: 48-layer CNN for additional frame features.

    3. Temporal Modeling: Incorporates GRU (Gated Recurrent Unit) to capture the sequential dependencies and temporal dynamics within the video.

    4. Frame Handling: Addresses varying numbers of frames in videos by converting them into a 3D tensor representation and padding with zeros if the number of frames is less than the maximum allowed.

    5. Steps Involved:

    Capture video frames.

    Extract frames until the maximum frame count is reached.

    Pad with zero frames if necessary to standardize frame count.

  • Project 5

    Human Activity Recognition

    The inference pipeline consists of both the Detectron2 model and a custom LSTM model.

    1. Video Input Processing: Our application accepts video input, iterates through the frames, and uses Detectron2 for keypoint detection on each frame.

    2. Buffer Management: Keypoint results are appended to a buffer of size 32, operating in a sliding window fashion.

    3. Action Identification: Contents of the buffer are sent to the trained LSTM model for action identification.

    4. Web Application: The Flask-based web application provides a UI to accept video input from the user.

    5. Action Annotation: Actions detected by the inference pipeline are annotated on the video and displayed as the result.
